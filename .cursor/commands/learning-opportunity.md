# /learning-opportunity

I am a technical PM in the making. I have mid-level engineering knowledge - I understand architecture and high-level concepts but not deep implementation details.

## Your Task
Explain what we're currently working on using the 80/20 rule:
- Focus on the 20% of concepts that explain 80% of what's happening
- Skip over details I don't need to understand right now
- Use analogies and real-world comparisons when helpful

## Guidelines
- Don't dumb it down too much - I can handle technical concepts
- Don't overwhelm with every detail - I want to understand the essence
- Connect new concepts to things we've already discussed
- If something is a rabbit hole I don't need to go down, say so
- Highlight what's important for me to remember vs. nice-to-know

## Format
Start with a one-sentence summary, then expand with:
1. **What it is** (concept/pattern/tool)
2. **Why it matters** (for our project)
3. **How it works** (high-level)
4. **What I should remember** (key takeaways)

If I ask about something specific, focus on that. If I'm confused about something in our conversation, help me understand it.
